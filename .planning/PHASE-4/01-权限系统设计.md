# Phase 4 权限系统设计 / Permission System Design

**创建时间:** 2026-03-26
**状态:** 已规划，待实施
**阶段:** Phase 4

---

## 1. 背景与目标 / Background & Goals

### 1.1 现状问题

当前系统只有单一管理员角色，无法满足学校多级管理需求：
- 校级管理员管理全校
- 部门管理员管理本部门
- 普通用户仅提交本人证书

### 1.2 目标

实现三级权限体系，支持：
- 部门动态管理
- 权限分级控制
- 批量用户操作
- 审核流程管理

---

## 2. 权限体系 / Permission System

### 2.1 三级角色定义

| 角色 | 英文 | 说明 |
|------|------|------|
| 校级管理员 | school_admin | 全校管理，最高权限 |
| 部门管理员 | dept_admin | 本部门管理 |
| 普通用户 | user | 仅提交本人证书 |

### 2.2 角色层级关系

```
校级管理员 (school_admin)
    └── 部门管理员 (dept_admin)
            └── 普通用户 (user)
```

### 2.3 关键约束

| 约束项 | 说明 |
|--------|------|
| 部门创建 | 仅校级管理员 |
| 用户归属 | 一用户一部门 |
| 部门管理员创建用户 | 仅本部门普通用户 |
| 审核流程 | 部门管理员直接通过，校级不干预部门业务 |

## 3. 部门管理 / Department Management

### 3.1 Department 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| name | String(100) | 部门名称 |
| code | String(50) | 部门代码（唯一） |
| description | Text | 描述 |
| status | Enum | active/inactive |
| created_at | DateTime | 创建时间 |
| created_by | ForeignKey(User) | 创建人 |

### 3.2 部门管理功能

| 功能 | 路径 | 方法 | 权限 |
|------|------|------|------|
| 部门列表 | /admin/departments | GET | 校级管理员 |
| 创建部门 | /admin/departments/new | GET/POST | 校级管理员 |
| 编辑部门 | /admin/departments/{id}/edit | GET/POST | 校级管理员 |
| 删除部门 | /admin/departments/{id}/delete | POST | 校级管理员 |
| 禁用部门 | /admin/departments/{id}/toggle | POST | 校级管理员 |
| 部门用户统计 | /admin/departments/{id}/stats | GET | 校级管理员 |

### 3.3 部门管理页面

```
部门管理
├── 部门列表
│   ├── 搜索（名称/代码）
│   ├── 筛选（状态）
│   └── 批量操作
├── 创建部门
│   └── 表单：名称、代码、描述
└── 编辑部门
    └── 表单：名称、代码、描述、状态
```

## 4. 用户-部门关联 / User-Department Relation

### 4.1 User 模型扩展

| 字段 | 类型 | 说明 |
|------|------|------|
| department_id | ForeignKey | 所属部门 |
| role | Enum | school_admin/dept_admin/user |
| must_change_password | Boolean | 首次登录必须改密 |
| created_by | ForeignKey | 创建人 |

### 4.2 注册流程

1. 用户注册时选择部门（下拉列表）
2. 部门管理员创建用户时自动归属本部门

## 5. 权限装饰器 / Permission Decorators

| 装饰器 | 用途 | 对象 |
|--------|------|------|
| @login_required | 登录验证 | 所有已登录用户 |
| @school_admin_required | 校级管理员专用 | 校级管理员 |
| @dept_admin_required | 部门管理员专用 | 部门管理员 |
| @admin_required | 管理员（校级+部门） | 校级和部门管理员 |
| @same_department | 同部门访问控制 | 内容访问 |

## 6. 批量操作 / Batch Operations

### 6.1 用户管理批量操作

| 功能 | 路径 | 方法 | 权限 |
|------|------|------|------|
| 批量选择 | - | Checkbox | 校级管理员 |
| 批量设置部门 | /admin/users/batch-department | POST | 校级管理员 |
| 批量设置角色 | /admin/users/batch-role | POST | 校级管理员 |
| 批量重置密码 | /admin/users/batch-reset-password | POST | 校级/部门管理员 |
| 批量删除 | /admin/users/batch-delete | POST | 校级/部门管理员 |
| 批量导入 | /admin/users/import | POST | 校级管理员 |

### 6.2 批量导入流程

```
下载模板 → 上传Excel → 预览确认 → 执行导入
```

**Excel 模板字段：**
| 字段 | 说明 |
|------|------|
| username | 用户名 |
| real_name | 真实姓名 |
| email | 邮箱 |
| department | 部门代码 |
| role | 角色（user/dept_admin） |

**导入规则：**
- 默认密码：`123456`
- 首次登录强制改密：开启
- 角色为 user

## 11. 数据库迁移 / Database Migration

```sql
-- 部门表
CREATE TABLE departments (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    code VARCHAR(50) UNIQUE NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'active',
    created_at DATETIME,
    created_by INTEGER REFERENCES users(id)
);

-- 用户表扩展
ALTER TABLE users ADD COLUMN department_id INTEGER REFERENCES departments(id);
ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'user';
ALTER TABLE users ADD COLUMN must_change_password BOOLEAN DEFAULT TRUE;
```
